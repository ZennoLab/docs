# Переменные окружения ZennoPoster

## Описание

Помимо [❗→ пользовательских переменных](/wiki/spaces/RU/pages/486309922 "/wiki/spaces/RU/pages/486309922") существуют так же переменные окружения. В них хранятся различные параметры страницы (URL, DOM, текст, домен, текст нотификаций и алертов и т. д.), переменные времени и даты, переменные проекта (имя, папка, правила прокси, id последней ошибки и т. д.) , многочисленные переменные профиля (почта, пол, имя, юзерагент и т. д.).

Ниже Вы можете ознакомится со списком доступных переменных и описанием того, что хранится в каждой из них.

  

## Переменные текущей активной вкладки

<details>
<summary>Нажмите здесь, чтобы развернуть</summary>

| **Имя** | **Описание** |
| --- | --- |
| `{ -Page.Url- }` | Текущий URL страницы |
| `{ -Page.Source- }` | Исходный код текущей веб-страницы |
| `{ -Page.Dom- }` | DOM-модель текущей веб-страницы |
| `{ -Page.Text- }` | Текст текущей веб-страницы без HTML тегов |
| `{ -Page.FullDomain- }` | Полный домен текущей веб-страницы.<br />С учётом поддоменов. |
| `{ -Page.Domain- }` | Основной домен текущей веб-страницы.<br />Без учёта поддоменов. |
| `{ -Page.LastAlertTitle- }` | Заголовок алерта, выводимого на сайтах с помощью JS функции alert() |
| `{ -Page.LastAlertText- }` | Текст алерта, выводимого на сайтах с помощью JS функции alert() |
| `{ -Page.LastNotificationTitle- }` | Заголовок последнего уведомления от сайта<br /><br /><br />Речь о сообщениях, которые приходят после того, как Вы разрешаете сайту присылать Вам уведомления. |
| `{ -Page.LastNotificationText- }` | Текст последнего уведомления от сайта<br /><br /><br />Речь о сообщениях, которые приходят после того, как Вы разрешаете сайту присылать Вам уведомления. |

</details>
## Время и дата

<details>
<summary>Нажмите здесь, чтобы развернуть</summary>

| **Имя** | **Описание** |
| --- | --- |
| `{ -TimeNow.Date- }` | Текущие дата и время в формате `ММ/ДД/ГГГГ чч:мм:сс` (пример - `01/25/2020 14:41:18`).<br /><br /><br /><br />Вы можете менять формат выводимой строки! Для этого перед закрывающей скобкой надо добавить шаблон формата. <br /><br />Пример: `{-TimeNow.Date dd.MM.yyyy HH.mm-}`, после выполнения получим такую строку - `25.01.2020 14.41`<br /><br />Подробнее о форматировании можно прочитать [тут](https://msdn.microsoft.com/ru-ru/library/8kb3ddd4%28v=vs.100%29.aspx "https://msdn.microsoft.com/ru-ru/library/8kb3ddd4(v=vs.100).aspx") и [тут](https://msdn.microsoft.com/ru-ru/library/az4se3k1%28v=vs.100%29.aspx "https://msdn.microsoft.com/ru-ru/library/az4se3k1(v=vs.100).aspx"). |
| `{ -TimeNow.Day- }` | День месяца |
| `{ -TimeNow.Month- }` | Текущий месяц (числом) |
| `{ -TimeNow.Year- }` | Текущий год |
| `{ -TimeNow.DayOfWeek- }` | День недели, на английском. |
| `{ -TimeNow.Hour- }` | Текущий час (в 24 часовом формате) |
| `{ -TimeNow.Minute- }` | Текущая минута |
| `{ -TimeNow.Second- }` | Текущая секунда |
| `{ -TimeNow.Millisecond- }` | Текущая миллисекунда |
| `{ -TimeNow.TimeNow- }` | Текущие дата и время в формате `ГГГГ-ММ-ДД чч-мм-сс--миллисекунды` (`2020-01-25 14-41-59--834`)<br /><br /><br />**Не** поддерживает форматирование! |
| `{ -TimeNow.UnixTime- }` | [Unix-время](https://ru.wikipedia.org/wiki/Unix-%D0%B2%D1%80%D0%B5%D0%BC%D1%8F "https://ru.wikipedia.org/wiki/Unix-%D0%B2%D1%80%D0%B5%D0%BC%D1%8F"). Формат - `1629891362.23929` |

</details>

## Системные переменные (Environment)

<details>
<summary>Нажмите здесь, чтобы развернуть</summary>

| **Имя** | **Описание** |
| --- | --- |
| `{ -Environment.CurrentUser- }` | ID пользователя ZennoPoster, из личного кабинета, работающего с шаблоном. |
| `{ -Environment.TickCount- }` | Количество миллисекунд прошедших с момента включения компьютера.<br /><br /><br />Эта переменная основана на [Environment.TickCount из C#](https://docs.microsoft.com/en-us/dotnet/api/system.environment.tickcount "https://docs.microsoft.com/en-us/dotnet/api/system.environment.tickcount"), поэтому ей присуща одна особенность: если компьютер будет долго и беспрерывно работать, то через ~24,9 дней эта переменная достигнет максимального положительного значения - `2147483647`. После этого перепрыгнет к минимально возможному значению - `-2147483648` и начнёт отсчёт к нулю (достигнет его через ~24,9 дней). После достижения нуля снова начнётся увеличение счётчика до максимально возможного значения (`2147483647`) и так по кругу. |

</details>
