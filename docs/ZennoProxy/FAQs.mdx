## Вопрос - ответ

### **Не работают подключения. Что делать?** 

Если подключение к прокси не устанавливается (в том числе при попытке из России) или есть затруднения из\-за скорости работы прокси, смените зеркало в списке подключения. Подробнее в разделе \*«Работа со списками»\*.

### **Что такое параллельность и как не нарваться на блокировку?** 

Параллельность – число одновременных потоков через один IP. Чем выше, тем быстрее сбор, но тем выше риск капчи/бана. Для CG-NAT (мобильные) держите ≤5-10 потоков; для резидентных можно 20-50, если IP не многопользовательский.

### **Как избежать «выгорания» IP-адресов?**

Ставьте лимит запросов на IP, соблюдайте паузы (throttling), меняйте строку User-Agent, чистите cookie. При появлении капчи – временно убирайте IP из пула или переведите его в резерв.

### **Что такое таргетинг в прокси?**

Это возможность выбирать IP по атрибутам: страна, город, оператор (ASN). Нужен, чтобы показывать сайтам «правдоподобного» клиента из нужного региона (например, цены локального магазина).

### **Какой тип подключения мне выбрать: резидентный или мобильный?** 

### **Резидентные прокси**   
Идеальны для: массового стабильного скрейпа, одновременных потоков 20-50+, долгих «липких» сессий, задач с гео-таргетингом «до города».

+ Ниже пинг и выше пропускная способность → быстрее и предсказуемее ответы.  
+ IP держится стабильнее, поэтому проще строить статические цепочки и кэшировать DNS.  
+ Реже используется CG-NAT, поэтому один IP часто «почти ваш» (меньше соседей → меньше капч).

- Анти-боты могут давать чуть меньше кредита доверия, чем мобильным IP.  
- При слишком агрессивном скрейпе возможны баны / rate-limit, если пул мал.

### **Мобильные прокси**  
Идеальны для: строгих антибот-проверок (регистрация, логин, прогрев аккаунтов, PWA/моб.-приложения), когда нужен мобильный ASN и «живой» отпечаток смартфона.

+ Максимальный уровень «человечности» – сайты видят реальный мобильный ASN, что повышает доверие.  
+ Терпимее к «подозрительным» действиям (частые логины, смена страниц), т.к. мобильные сети априори шумнее.

- CG-NAT → десятки пользователей делят один IP: сильная параллельность (10+ потоков) повышает риск капч для всех «соседей».  
- Пинг выше, возможны скачки скорости и редкие «переподключения» с внезапной сменой IP.  
- Чаще «многопользовательские», поэтому нет гарантии эксклюзивности.
