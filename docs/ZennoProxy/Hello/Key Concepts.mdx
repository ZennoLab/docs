### **ZennoProxy**  
– сервис для быстрого создания прокси.

### **Прокси (прокси-сервер)**  
– промежуточный узел между вашим устройством (клиентом) и сервером назначения.  

### **Подключение через прокси**  
– способ подключения, при котором ваш трафик идет не напрямую к сайту/сервису, а сперва на промежуточный сервер (прокси), и уже он делает запрос к целевым ресурсам от своего имени с возможностью маскировать IP/гео, применять правила доступа, кэшировать и маршрутизировать трафик. 

:::note **Примечание**
Не путать с VPN. VPN шифрует и туннелирует весь трафик устройства; прокси чаще работает на уровне конкретного приложения/протокола.
:::

### **IP-адрес прокси**  
– IP-адрес, который используется прокси-сервером для представления пользователя в сети.

### **Статическая VS Ротационная сессия**  
Статические и ротационные сессии определяют, как IP-адрес используется во время взаимодействия с сайтами. Статическая сессия предполагает использование одного и того же IP-адреса на протяжении всего сеанса, в то время как ротационная сессия подразумевает смену IP-адреса на каждом новом запросе или через определенные промежутки времени. 

### **Липкая (sticky) сессия**  
– это режим, при котором «распределитель запросов» (балансировщик) перед пулом серверов после первого вашего запроса выбирает конкретный сервер, запоминает этот выбор (обычно по cookie/ID сессии или хешу IP) и направляет все дальнейшие запросы этой сессии только на него, пока сессия активна или не истечет таймаут.

*Липкая и статическая сессия это одно и то же?* 

Обе эти концепции описывают ситуацию, когда пользователь в рамках одного сеанса связи с сервером всегда направляется на один и тот же сервер или узел в кластере. Это достигается путем сохранения информации о сессии и привязке ее к конкретному серверу. 

Разница может заключаться в терминологии в разных контекстах. В некоторых случаях "статическая сессия" может подразумевать более общее понятие сохранения состояния, не обязательно связанное с балансировкой нагрузки, в то время как "липкая сессия" чаще используется в контексте балансировки нагрузки для обозначения привязки сессии к определенному серверу. 

### **Резидентные и мобильные прокси**  
У резидентных прокси трафик идёт через сети домашних интернет-провайдеров. По базам и ASN выглядят как обычные квартиры/роутеры.  
У мобильных прокси трафик идёт через сети сотовых операторов. Часто десятки устройств делят один внешний IP из\-за CG-NAT, поэтому такой IP «похож на смартфон с LTE/5G».

Сравнение резидентных и мобильных прокси

|  | Резидентные | Мобильные |
| :---- | :---- | :---- |
| источник IP / ASN | домашние сети  | сотовые операторы  |
| доверие антиботов | «обычный пользователь ПК/роутера» | «живой смартфон», обычно получают больше кредита доверия |
| скорость и пинг | обычно быстрее и стабильнее | выше пинг, возможны скачки скорости |
| стабильность IP (липкость) | легко держать статические сессии на 10–60 мин и дольше | статические сессии есть, но возможны редкие «скачки» IP у оператора (передподключения, балансировка) |
| shared-характер | нередко почти «вы одни» за IP | почти всегда shared (CG-NAT) – десятки/сотни пользователей за одним IP |
| параллельность | терпят больше одновременных потоков | из\-за CG-NAT слишком агрессивная параллельность повышает риск капч/блоков для всех «соседей» по IP |
| географическая точность | город/регион совпадают стабильнее | гео иногда «прыгает» между узлами оператора |

### **Параллельность**  
– сколько запросов/потоков идут одновременно через один IP/аккаунт/процесс.

### **Пул**  
Пул прокси – это набор подготовленных IP-адресов (точек выхода). Из пула сервис выдает один или несколько адресов: фиксированные для статических сессий или разные при ротационных сессиях.

### **Ключевые характеристики пула**  
Размер и разнообразие: сколько IP и насколько они разные по подсетям, провайдерам (ASN) и гео.  
Тип: резидентные / мобильные.  
Таргетинг: общий, по стране/городу, по ASN.  
Эксклюзивность: общий (shared) или выделенный (dedicated/private).  
Политика выдачи: ротация по времени/по запросу/по ошибке, «липкие» сессии и их TTL.  
Лимиты: параллельные потоки, окно повторного использования (cooldown), защита от выгорания IP.

Иногда говорят о пуле портов (много портов, каждый – динамически назначаемый IP из большого пула) или о пуле сессий на стороне клиента – но в прокси-контексте под «пулом» почти всегда подразумевают именно набор выходных IP.

### **Таргетинг**  
– в контексте прокси-серверов, «таргетинг» означает выбор конкретного прокси-сервера или группы прокси-серверов для использования в определенных целях. Это может включать в себя выбор сервера по географическому местоположению (страна, город), типу сети (мобильный, домашний) или другим критериям, чтобы обеспечить желаемое поведение при работе с интернет-ресурсами. 

### **Антифрод система**  
– программный софт, который оценивает риск операции (логин, платёж, регистрация, комментарий и т.п.) и принимает решение: разрешить, запросить дополнительную проверку или отклонить операцию. Широко применяется в финтех/банках, маркетплейсах, соцсетях, игровой индустрии.

### **Какие сигналы анализирует:**

* Сеть: IP/ASN, страна/город, прокси/VPN/DC, скорость/стабильность, частота смены IP.  
* Устройство/браузер: User-Agent, отпечаток (Canvas/WebGL/Fonts), часовой пояс, язык, WebRTC/DNS.  
* Поведение: скорость печати, паттерны кликов/скролла, траектории мыши, «человечность» сессии.  
* История: репутация аккаунта, частота запросов, «velocity» (сколько действий за окно).  
* Бизнес-правила: лимиты по товарам/цене/стране, чёрные/серые списки, аномальные комбинации.  
* Платежка: 3-DS/SCA, несостыковки карты/гео/устройства, chargeback-риск.

В итоге система считает баллы риска и принимает одно из трёх решений:   
– пропустить,   
– попросить дополнительное подтверждение (решить задачу «я не робот», ввести код из сообщения на телефон, подтвердить письмо на электронную почту, ввести код из приложения/токена – любой из доступных способов двухэтапной проверки),   
– отклонить.

### **HTTP / SOCKS5**

|  | HTTP-прокси | SOCKS5-прокси |
| :---- | :---- | :---- |
| Назначение | проксирование HTTP;  HTTPS через метод CONNECT (туннель TCP до цели) | универсальный транспортный прокси на уровне сокетов |
| Возможности | кэширование, заголовки, базовая аутентификация (Proxy-Authorization) | TCP и UDP (UDP ASSOCIATE), DNS-прокси у прокси-сервера, логин/пароль |
| Ограничения | только TCP;  UDP/QUIC (HTTP/3) не проходит обычным образом | подходит для не-HTTP протоколов (например, некоторые мессенджеры/игры), лучше дружит с QUIC/UDP (если включён UDP на стороне провайдера) |
| Типичные порты | 3128, 8080 (может быть любой) | 1080 |

### **Что выбрать:**

Браузерный веб / скрапинг сайтов / API по HTTPS → HTTP-прокси .

Нужен UDP / нестандартные протоколы / совместимо с HTTP/3 → SOCKS5-прокси.

### **Скрейп (скрейпить, скрапинг, web scraping)**  
– автоматический сбор данных с веб\-страниц или API, включая загрузку, при необходимости рендер, извлечение и сохранение. Например, сбор каталогов, цен, отзывов, рейтингов, новостей.

### **Парсинг**   
–  разбор полученного HTML/JSON/XML и извлечение целевых полей (цена, название, рейтинг и т. п.).

### **Скрейп VS Парсинг**  
Парсинг – это этап внутри скрепа. Результат парсинга – уже структурированные данные (таблица/JSON/БД), полученные из сырого контента. Иными словами:  
скрейп – добываем и обрабатываем данные (включает парсинг как один из шагов).  
парсинг – разбираем уже полученный контент и вынимаем нужные поля.

Можно скрейпить без парсинга (скачать HTML «впрок», разобрать позже).  
Можно парсить без скрейпа (вам дали файл HTML/JSON – вы его просто разобрали).

### **ASN**  
– уникальный ID автономной системы, управляемой одной организацией и имеющей единую маршрутизацию; по нему системы узнают друг друга при обмене маршрутами.   
К автономным системам относятся: интернет-провайдеры и мобильные операторы, облачные платформы, хостинг-провайдеры и дата-центры, CDN, крупные корпоративные, университетские и государственные сети, узлы обмена трафиком IX/IXP.

### **NAT**  
– Network Address Translation – подмена сетевых адресов на границе сети. Чаще всего много внутренних (частных) адресов выходят в интернет через один или несколько публичных IP.

### **CG-NAT (Carrier-Grade NAT)**   
– NAT на стороне оператора: десятки/сотни абонентов делят один публичный IPv4. Широко используется в мобильных сетях и у части фиксированных провайдеров.
