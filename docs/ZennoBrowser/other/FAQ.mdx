## **Частые вопросы**

— Оплатить дополнительные потоки вы можете [в личном кабинете](https://account.zennolab.com/personal-area-zenno-browser/update-tariff), во вкладке “ZennoBrowser”. В ней будет информация о доступных тарифах:

![FAQ1](..\other\assets\FAQ\FAQ1.png)

— Если появляется ошибка “Выполнение действия WebBrowserSettings Ошибка запуска профиля ZennoBrowser (16F092BE-3196-4F94-89B3-885DF9FE64AC): Can't get correct workspace ```{WorkspaceId}```”, то поможет обновление на актуальную версию ЗП 7.8.2.0.

— Интеграция ZB и ZP7 позволит при массовом создании профилей назначать для каждого свои прокси.

— Фильтрация профилей будет добавлена в грядущих релизах.

— Лимита на созданные профили на стороне ZB нет, их максимальное количество определяется доступным местом на жёстких дисках ПК.

— Папки с профилями хранятся по адресу: C:\Users\{User}\AppData\Local\ZennoLab\Profiles Также её можно переназначить на другой диск. Для этого нужно добавить файл zp8.profilemanager.json в папку “Документы” с наполнением:

``` json
{
 "Paths": {
   "ProfilesRootPath": "C:/Users/Administrator/Desktop/TWST"
}
}
```

В файле нужно указать новый путь для профилей.

Механизма импорта или экспорта пока что в программе нет, но мы планируем добавить в будущем.

— Чтобы не терять созданные данные, можно выбрать опцию “Не удалять профили” при переустановке.

![FAQ2](..\other\assets\FAQ\FAQ2.png)

— К ZennoBrowser можно подключать 2 и более ZennoPoster, установленных на компьютер, при условии, если один пользователь.

— Интеграция работает, если в ZennoBrowser и ZennoPoster разные профили.

— Пока что нет прямой интеграции ZB и ZP.

— Мобильные профили для Android и iOS пока что недоступны в ZB.

— Вручную заполнить UserAgent не получится, можно работать только с предложенными параметрами.
