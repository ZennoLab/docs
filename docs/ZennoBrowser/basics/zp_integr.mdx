## **Запуск и настройка**


**ZennoPoster 7** открывает новые горизонты автоматизации благодаря интеграции с **ZennoBrowser** — профессиональным антидетект-браузером.

### **Ключевые преимущества интеграции**

**Основные возможности:**

 - Управление профилями в режиме реального времени;
 - Работа с прокси на продвинутом уровне;
 - Изолированные среды для каждого процесса;
 - Контроль цифровых отпечатков.

**Основные функции:**

 - Создание профилей с уникальными настройками;
 - Запуск браузерных инстансов и управление ими; 
 - Удаление профилей без потери данных.


**Процесс интеграции обеспечивает:**

 - Бесшовное взаимодействие между продуктами;
 - Сохранение интерфейса ZennoPoster 7;
 - Расширение возможностей за счет функций ZennoBrowser.


### **Процесс настройки**

**Шаг 1.** Запустите ProjectMaker:

![25](..\basics\assets\Integration\25.png)

**Шаг 2.** В поисковой строке окна “Действия” введите слово “Интеграция”. Появятся варианты действий:

![26](..\basics\assets\Integration\26.png)

**Шаг 3.** Запускаем инстанс ZennoBrowser и приступаем к работе.

![27](..\basics\assets\Integration\27.png)


### **Какие возможности даёт интеграция?**

#### **1. Получение профилей ZennoBrowser**

Действие "Получить профили ZennoBrowser" предназначено для извлечения информации о существующих профилях браузера из системы ZennoBrowser.

![28](..\basics\assets\Integration\28.png)

**Принцип работы** 

Действие выполняет запрос к ZennoBrowser и возвращает полную информацию о всех доступных профилях в виде структурированных данных.

**Формат результата**

Результат выполнения представляет собой JSON-массив объектов, где каждый объект содержит детальную информацию об отдельном профиле:
 - ID профиля;
 - Имя профиля;
 - Настройки браузера; 
 - Статус профиля;
 - Данные прокси;
 - Другие метаданные.

#### **2. Обработка результатов**

Для корректной работы с полученными данными рекомендуется использовать действие "Парсинг JSON", которое позволит:
 - Извлечь конкретные параметры профилей;
 - Отфильтровать профили по заданным критериям;
 - Преобразовать данные в удобный для работы формат.

#### **3. Обработка ошибок**

В случае проблем с подключением к ZennoBrowser (недоступность сервиса, неверные настройки соединения) действие вернёт соответствующую ошибку, которую необходимо обработать в логике проекта.

#### **4. Сохранение результата**

Укажите переменную для сохранения JSON-ответа в поле "Поместить в переменную" для дальнейшего использования в проекте.

**Для наглядности мы подготовили [образец проекта](https://drive.google.com/drive/u/0/folders/1JsWGyuzFng-E61bDG7FleagolyB4pDRT) с уже собранными кубиками, позволяющими на выходе получить список профилей.**



## **Интеграция с действием "Запустить инстанс"**

Это действие предназначено для запуска браузера с использованием конкретного профиля ZennoBrowser по его идентификатору.

![29](..\basics\assets\Integration\29.png)
![30](..\basics\assets\Integration\30.png)

### **Принцип работы**

Действие принимает ID профиля ZennoBrowser и инициирует запуск браузерного инстанса с соответствующими настройками профиля, включая:
 - Цифровые отпечатки браузера;
 - Настройки прокси;
 - Пользовательские данные;
 - Расширения и конфигурации.

### **Настройка профиля**

При первом открытии настроек действия автоматически загружается актуальный список профилей из ZennoBrowser. Доступны два способа указания профиля:

 1. **Ручной ввод ID:** введите идентификатор профиля в соответствующее поле.

 2. **Выбор из списка:**

 -> Сначала выберите папку профилей из выпадающего списка.

 -> Затем выберите конкретный профиль внутри выбранной папки.

### **Обновление списка профилей**

Для получения актуальной информации о доступных профилях используйте кнопку "Обновить" в настройках действия. Это особенно полезно при добавлении новых профилей в ZennoBrowser.

![31](..\basics\assets\Integration\31.png)

### **Новая опция браузера**

В стандартное действие "Запустить инстанс" добавлен новый тип браузера — "Chromium from ZennoBrowser (beta)".

![32](..\basics\assets\Integration\32.png)

Эта интеграция обеспечивает единообразный подход к запуску различных типов браузеров в рамках одного действия.


## **Создание профилей ZennoBrowser**

Действие "Создание профилей ZennoBrowser" предоставляет два метода создания новых профилей браузера:

### **1. Создание профилей по умолчанию**

При выборе режима "По умолчанию" профили создаются с оптимизированными настройками. Необходимо указать:
 - Количество профилей — требуемое количество создаваемых профилей;
 - Настройки прокси — параметры прокси-серверов для профилей.

 ![33](..\basics\assets\Integration\33.png)

### **2. Создание на основе профиля-донора**

При выборе режима "Использовать данные профиля" новые профили наследуют все настройки указанного существующего профиля-донора:
 - Укажите ID профиля-донора или выберите его из списка;
 - Будут скопированы настройки отпечатков браузера, расширения и другие параметры конфигурации.

**Важно:** копируются настройки отпечатков, а не сами отпечатки — каждый созданный профиль получит уникальные отпечатки в соответствии с настройками донора

![34](..\basics\assets\Integration\34.png)

### **Настройки прокси**

Система поддерживает гибкое управление прокси-серверами:
 - Указать список прокси — выберите готовый список из ZennoBrowser;
 - Максимальное количество профилей на прокси — ограничение для предотвращения перегрузки прокси;
 - Исключение повторений — каждый прокси используется только один раз при создании профилей.

**Сохранение результатов**

Данные созданных профилей обязательно должны быть сохранены для дальнейшего использования:
 - В список — для работы с множественными профилями;
 - В переменную — для сохранения информации об одном профиле.


## **Удаление профилей ZennoBrowser**

Действие "Удалить профили ZennoBrowser" предназначено для безвозвратного удаления выбранных профилей браузера из системы ZennoBrowser.

![35](..\basics\assets\Integration\35.png)

### **Способы указания профилей для удаления**

 1. **Ручной ввод ID:** укажите идентификатор профиля в поле "Установите ID профилей".

Поддерживается использование переменных для динамического указания ID.

 2. **Выбор из списка:**
 - Выберите папку профилей из выпадающего списка;
 - Выберите конкретный профиль для удаления из выбранной папки.

### **Обновление списка профилей**

Для получения актуального списка доступных профилей используйте кнопку "Обновить" в настройках действия.


## **Особенности интеграции ZennoBrowser**

### **Работа с браузером**

Интегрированный браузер функционирует в ZennoPoster7 аналогично другим, то есть действия работы со страницами и элементами браузера работают нативно.
Если у вас не установлен ZennoBrowser, то действие запуска инстанса предложит вам подать заявку на тестирование нового продукта, чтобы вы смогли установить его.

![36](..\basics\assets\Integration\36.png)

### **Запуск и подключение**

Для работы интеграции необязательно, чтобы ZennoBrowser был запущен. ZennoPoster7 сможет запустить его самостоятельно в фоновом режиме, без демонстрации окна ZennoBrowser. Однако, если ZennoBrowser будет запущен предварительно, получение профилей или запуск нового браузера выполнятся быстрее.
Если ZB уже был запущен, то есть на экране есть интерфейсное окно, при запуске профиля “ЗанноПостером” в ZennoBrowser будет отражаться его состояние и будет доступна кнопка Stop. Вы можете остановить браузер и освободить профиль в любой момент, но это может привести к ошибкам в работе проекта ZennoPoster7.

При фоновом запуске профилей из интеграции окно с ZB не открывается, но можно запустить браузер с помощью ярлыка, и он покажет актуальное состояние профилей.

В случае если ZennoPoster7 не смог подключиться к ZennoBrowser, новые действия вернут ошибку. Также ZennoBrowser может отклонить запуск браузера по некоторым причинам: например, если профиль уже занят, это тоже приведет к ошибке действия.

### **Различия в ProjectMaker и ZennoPoster**

При запуске нового браузера в ProjectMaker, браузер нативно встраивается в UI и вы можете пользоваться всеми доступными инструментами ProjectMaker. При этом, как и в браузере Chromium, для ProjectMaker отключено использование GPU. 

### **Работа с профилями**

При работе с новым браузером применяются профили и эмуляции ZennoBrowser. Вся работа эмуляций и настроек браузера ZP7 и работа с профилем ZP7 игнорируется. При этом соответствующие действия выполняются без ошибок, но не производят никаких изменений. Все действия, которые могут повлиять на профиль игнорируются.

### **Управление окном браузера**

В ZennoPoster7 при показе инстанса отображается встроенный браузер. Кнопки свернуть/закрыть работают как стандартные кнопки окна и могут привести к непредвиденному поведению браузера, поэтому не рекомендуется их использовать. Для удобства работы с реальным браузером в ZennoPoster в окне превью инстансов было расширено контекстное меню. В будущем планируется сделать работу кнопок свернуть/закрыть аналогично с другими типами браузеров в ZennoPoster7.


## **Контекстное меню превью инстанса**

Для удобства работы с окнами Chromium из ZennoBrowser контекстное меню в ZennoPoster7 расширено дополнительными функциями управления.

### **Пункты меню:**
**Показать** — отображает окно браузера, если оно было скрыто.
**Скрыть** — скрывает окно браузера без завершения его работы.
**Продолжить** — используется для продолжения выполнения при "Ожидании действий пользователя".

![37](..\basics\assets\Integration\37.png)

### **Преимущества**

Расширенное контекстное меню позволяет управлять всеми ключевыми функциями ZennoPoster прямо из интерфейса превью инстансов, обеспечивая полный контроль над браузерами ZennoBrowser без необходимости переключения между различными окнами приложения.
