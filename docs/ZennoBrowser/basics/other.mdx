## **Примеры использования продукта**

**ZennoBrowser** поможет вам с задачами, сопряжёнными с созданием аккаунтов на любом сервисе (криптосервис, почтовый клиент, хостинг, форумы, маркетплейсы и так далее). Он подходит под все схемы работы с массовыми действиями.

Наш антидетект-браузер позволяет в ручном режиме работать с рутинными действиями. Интеграция с ZennoPoster открывает возможности автоматизации этих действий и перехода на другой уровень масштабирования. Соответственно, ручные действия в ZennoBrowser служат для решения точечных задач, а автоматизация даёт ресурсы для управления процессами при больших объёмах работ.

С конкретными примерами можно ознакомиться в наших конкурсных разделах [на форуме](https://zenno.club/discussion/forums/poleznye-stati-istorii-uspexa.154). Все решения, которые предлагают наши участники, можно использовать и в ручном режиме, и в автоматическом. Выбор зависит только от ожидаемых объёмов и результатов.


## **Частые вопросы**

— Оплатить дополнительные потоки вы можете [в личном кабинете](https://account.zennolab.com/personal-area-zenno-browser/update-tariff), во вкладке “ZennoBrowser”. В ней будет информация о доступных тарифах:

![38](..\basics\assets\Other\38.png)

— Если появляется ошибка “Выполнение действия WebBrowserSettings Ошибка запуска профиля ZennoBrowser (16F092BE-3196-4F94-89B3-885DF9FE64AC): Can't get correct workspace ```{WorkspaceId}```”, то поможет обновление на актуальную версию ЗП 7.8.2.0.

— Интеграция ZB и ZP7 позволит при массовом создании профилей назначать для каждого свои прокси.

— Фильтрация профилей будет добавлена в грядущих релизах.

— Лимита на созданные профили на стороне ZB нет, их максимальное количество определяется доступным местом на жёстких дисках ПК.

— Папки с профилями хранятся по адресу: C:\Users\{User}\AppData\Local\ZennoLab\Profiles Также её можно переназначить на другой диск. Для этого нужно добавить файл zp8.profilemanager.json в папку “Документы” с наполнением:

``` json
{
 "Paths": {
   "ProfilesRootPath": "C:/Users/Administrator/Desktop/TWST"
}
}
```

В файле нужно указать новый путь для профилей.

Механизма импорта или экспорта пока что в программе нет, но мы планируем добавить в будущем.

— Чтобы не терять созданные данные, можно выбрать опцию “Не удалять профили” при переустановке.

![39](..\basics\assets\Other\39.png)

— К ZennoBrowser можно подключать 2 и более ZennoPoster, установленных на компьютер, при условии, если один пользователь.

— Интеграция работает, если в ZennoBrowser и ZennoPoster разные профили.

— Пока что нет прямой интеграции ZB и ZP.

— Мобильные профили для Android и iOS пока что недоступны в ZB.

— Вручную заполнить UserAgent не получится, можно работать только с предложенными параметрами.


## **Обновление**

ZennoBrowser всегда устанавливается в своей актуальной версии. Когда появляется обновление, нужно перейти в программу, нажать на значок своего пользовательского профиля в левом нижнем углу экрана и в “Общих настройках” запустить проверку апдейтов: тогда появится доступный вариант, и ZennoBrowser обновится.

Также есть важная фича: при обновлении браузера также обновятся и версии профилей, соответственно. У обновлённых профилей всегда будет актуальная версия ядра браузера. 

![40](..\basics\assets\Other\40.png)

О выходе обновлений мы всегда рассказываем [в нашем Telegram-канале](https://t.me/zennolab) и [на форуме](https://zenno.club/discussion/forums/zennobrowser.340/) 
Также о выпуске свежей версии вы узнаете из самого браузера, когда в нём появится окно с оповещением о её появлении.
